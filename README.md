# Windsurf Manager 用户指南

## 📖 简介

Windsurf Manager 是一个 Windsurf 账号管理工具，帮助您集中管理多个 Windsurf 账号，实时查看使用情况、配额统计和账号状态。
有什么问题和建议都可以留言 我会修改的

## 🚀 快速开始

### 启动应用

双击 `Windsurf Manager.exe` 启动应用程序。

### 首次使用

1. 应用启动后会显示空状态页面
2. 点击"添加第一个账号"按钮开始添加账号

## 📊 主界面说明

### 1. 顶部统计栏

显示所有账号的汇总统计信息：

- **全部账号** (蓝色) - 已添加的账号总数
- **运行正常** (绿色) - Token 有效且正常工作的账号数
- **状态失效** (红色) - Token 已过期或无效的账号数
- **即将过期** (黄色) - 即将到期需要关注的账号数

### 2. 工具栏

位于统计栏下方，包含以下功能：

- **搜索框** - 输入账号名称或邮箱快速搜索
- **视图切换** - 在网格视图和列表视图之间切换
- **刷新全部** - 刷新所有账号的最新数据
- **添加账号** - 添加新的 Windsurf 账号

### 3. 账号卡片

每个账号以卡片形式展示，包含以下信息：

#### 卡片头部
- **账号名称** - 自定义的账号标识
- **邮箱地址** - 账号关联的邮箱
- **状态标签** - 显示账号当前状态（正常/失效/警告）

#### 使用情况
- **已用额度** - 当前已使用的 Credits 数量
- **进度条** - 可视化显示使用进度
  - 绿色：使用率 < 70%
  - 橙色：使用率 70-90%
  - 红色：使用率 > 90%
- **剩余额度** - 还可以使用的 Credits 数量

#### 底部信息
- **试用期** - 显示剩余试用天数
- **最后更新** - 数据最后刷新时间

#### 悬停操作
鼠标悬停在卡片上时，右上角会显示操作按钮：
- **👁️ 查看详情** - 查看账号完整信息和统计
- **🔄 刷新** - 刷新该账号数据
- **✏️ 编辑** - 修改账号信息
- **🗑️ 删除** - 删除该账号

## ➕ 添加账号

### 步骤 1: 获取 JWT Token

1. 打开浏览器访问 [windsurf.com](https://windsurf.com)
2. 登录您的 Windsurf 账号
3. 按 `F12` 打开开发者工具
4. 切换到 `Network` (网络) 标签
5. 刷新页面或进行任何操作
6. 在请求列表中找到任意 API 请求（通常是 `api.codeium.com` 开头）
7. 点击请求，在右侧 `Headers` (请求头) 中找到 `x-auth-token`
8. 复制完整的 Token 值

### 步骤 2: 获取 Refresh Token（可选）

如果需要自动刷新功能：

1. 在开发者工具中切换到 `Application` (应用) 标签
2. 展开左侧 `IndexedDB` → `firebaseLocalStorageDb`
3. 找到包含 `stsTokenManager` 的记录
4. 复制 `refreshToken` 字段的值

### 步骤 3: 填写账号信息

1. 点击"添加账号"按钮
2. 填写表单：
   - **账号名称** (可选) - 便于识别的名称，如"主账号"、"工作账号"
   - **JWT Token** (必填) - 粘贴步骤 1 获取的 Token
   - **Refresh Token** (可选) - 粘贴步骤 2 获取的 Token
3. 点击"保存"按钮

### 验证结果

- 添加成功后，账号卡片会自动出现在列表中
- 系统会自动获取账号信息和使用统计
- 如果显示"状态失效"，请检查 Token 是否正确

## 🔍 查看账号详情

点击账号卡片上的"👁️"图标，打开详情弹窗，包含：

### 基本信息
- 用户 ID
- 邮箱地址
- API Key
- 账号状态

### 使用统计
- **总配额** - 账号总共可用的 Credits
- **已使用** - 当前已消耗的 Credits
- **剩余额度** - 还可以使用的 Credits
- **使用进度条** - 可视化显示

### Cascade 统计
- **Memories** - 记忆功能使用次数
- **Terminal** - 终端命令执行次数
- **Web** - 网页搜索使用次数
- **MCP** - MCP 工具调用次数
- **Messages** - 消息总数
- **Credits** - 消耗的 Credits
- **Conversations** - 对话数量

### Cascade 模型使用
显示各个 AI 模型的使用情况：
- 模型名称（如 Claude Opus 4.5、Gemini 3 Pro）
- 使用百分比
- 消息数量
- 消耗 Credits
- 可视化进度条

### 时间信息
- 创建时间
- 最后更新时间
- 试用期剩余天数

## ✏️ 编辑账号

1. 点击账号卡片上的"✏️"图标
2. 在弹出的表单中修改信息
3. 点击"保存"保存更改

**注意：** 修改 Token 后会立即验证，如果无效会提示错误。

## 🗑️ 删除账号

1. 点击账号卡片上的"🗑️"图标
2. 确认删除操作
3. 账号将从列表中移除

**注意：** 删除操作不可恢复，请谨慎操作。

## 🔄 刷新数据

### 刷新单个账号
点击账号卡片上的"🔄"图标，刷新该账号的最新数据。

### 刷新全部账号
点击工具栏的"刷新全部"按钮，批量刷新所有账号数据。

**刷新频率建议：** 每次使用 Windsurf 后刷新一次，以获取最新的使用统计。

## 🔍 搜索功能

在工具栏的搜索框中输入关键词，可以快速筛选账号：
- 支持按账号名称搜索
- 支持按邮箱地址搜索
- 实时过滤，无需按回车

## 📱 视图模式

### 网格视图（默认）
- 以卡片形式展示账号
- 适合查看详细信息
- 每行显示 2-3 个账号

### 列表视图
- 以列表形式展示账号
- 适合快速浏览
- 信息更紧凑

点击工具栏的视图切换按钮在两种模式间切换。

## 🎨 状态说明

### 账号状态

- **🟢 正常** (绿色) - Token 有效，可以正常使用
- **🔴 失效** (红色) - Token 已过期或无效，需要重新获取
- **🟡 警告** (黄色) - 即将到期或使用率过高

### 使用率颜色

- **绿色** - 使用率 < 70%，额度充足
- **橙色** - 使用率 70-90%，需要注意
- **红色** - 使用率 > 90%，即将用完

- **—** 最小化窗口

## 💡 使用技巧

### 1. 账号命名建议
- 使用有意义的名称，如"主账号"、"测试账号"、"工作账号"
- 如果有多个账号，建议按用途分类命名

### 2. Token 管理
- Token 有效期通常为 1 小时
- 建议保存 Refresh Token 以便自动刷新
- 定期检查账号状态，及时更新失效的 Token

### 3. 配额监控
- 关注使用进度条颜色变化
- 使用率超过 70% 时考虑切换账号
- 定期刷新数据以获取准确的使用情况

### 4. 多账号管理
- 可以添加多个账号进行切换使用
- 通过搜索功能快速定位特定账号
- 使用列表视图可以一次查看更多账号

## ❓ 常见问题

### Q: Token 多久会过期？
A: JWT Token 通常 1 小时过期，Refresh Token 可以持续更长时间。建议添加 Refresh Token 以实现自动刷新。

### Q: 为什么显示"状态失效"？
A: 可能原因：
1. Token 已过期，需要重新获取
2. Token 格式不正确
3. 账号被封禁或注销
4. 网络连接问题

解决方法：重新获取 Token 并更新账号信息。

### Q: 数据多久更新一次？
A: 数据不会自动更新，需要手动点击刷新按钮。建议每次使用 Windsurf 后刷新一次。

### Q: 可以同时使用多个账号吗？
A: 可以。您可以添加多个账号，在不同的 Windsurf 客户端中使用不同的账号。

### Q: 删除账号会影响 Windsurf 官方账号吗？
A: 不会。删除操作只是从本地管理器中移除，不会影响您的 Windsurf 官方账号。

### Q: 如何备份账号数据？
A: 账号数据保存在应用程序目录的 `accounts.json` 文件中，可以手动备份该文件。

## 🔒 安全提示

1. **保护 Token 安全**
   - Token 相当于账号密码，请勿泄露给他人
   - 不要在公共场合展示包含 Token 的界面

2. **定期更新 Token**
   - 建议定期更换 Token 以提高安全性
   - 如果怀疑 Token 泄露，立即重新登录获取新 Token

3. **数据存储**
   - 账号数据存储在本地，请确保电脑安全
   - 建议定期备份 `accounts.json` 文件

4. **网络安全**
   - 应用使用 HTTPS 加密通信
   - 避免在不安全的网络环境下使用
